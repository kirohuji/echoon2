# Echoon2 待办需求清单

> 本文档列出当前项目已实现功能之外，仍有待开发的功能需求。
> **排除范围**：模考（Mock Exam）和练习模式（Practice Mode）核心功能已基本完备，不在此清单中。
> 优先级：P0 = 阻塞上线，P1 = 高优先级，P2 = 中优先级，P3 = 低优先级/锦上添花。

---

## 一、认证与账户安全

### 1.1 忘记密码 / 重置密码
- **优先级**：P0
- **描述**：用户通过邮箱或手机号找回密码的完整流程。
- **要点**：
  - 发送重置链接/验证码到邮箱或手机
  - 设置新密码页面
  - 密码强度校验
  - 重置成功后自动登录或引导到登录页
- **现状**：Better Auth 已配置 email/OTP 能力，但前端缺少对应页面。

### 1.2 注册时邮箱/手机验证
- **优先级**：P1
- **描述**：新用户注册时验证邮箱或手机号的有效性，防止虚假注册。
- **要点**：
  - 发送验证码 → 输入验证 → 完成注册
  - 验证码有效期与重发限制
- **现状**：Better Auth 支持 OTP，但前端注册页未集成验证流程。

### 1.3 账户注销
- **优先级**：P1
- **描述**：用户主动注销账户（含二次确认 + 数据清理提示）。
- **现状**：个人中心设置页有"注销账户"入口但无后续流程。

### 1.4 修改密码
- **优先级**：P1
- **描述**：已登录用户在账户设置中修改密码（需输入旧密码验证）。

## 二、支付与商业化

### 2.1 支付前端完整集成
- **优先级**：P0
- **描述**：前端对接支付宝/微信支付 SDK，完成从「选择套餐 → 创建订单 → 调起支付 → 支付结果」的完整闭环。
- **要点**：
  - 会员页套餐卡片点击"立即开通" → 创建订单 → 展示支付二维码/调起微信
  - 支付成功/失败的状态页面
  - 订单查询入口
- **现状**：后端 `pay` 模块已实现订单创建、状态查询、回调处理；前端会员页仅有静态展示。

### 2.2 订阅管理（自动续费）
- **优先级**：P2
- **描述**：支持自动续费订阅模式，用户可查看/取消自动续费。

### 2.3 优惠券/促销系统
- **优先级**：P3
- **描述**：支持优惠码、限时折扣、邀请奖励等营销手段。

---

## 三、内容管理（Admin）

### 3.1 题库内容 CRUD
- **优先级**：P0
- **描述**：管理后台支持对题库、专题、题目、题目内容的增删改查。
- **要点**：
  - 题目编辑器（中英文题目、答案、翻译、关键词、词汇）
  - 专题管理（景点介绍、导游规范等分类）
  - 题库配置（省份×语种×考试类型组合）
  - 批量导入/导出（Excel/CSV）
- **现状**：后端 `resource-library` 模块仅做文件管理；题目数据目前依靠 Prisma seed 手动填充。

### 3.2 模考试卷管理
- **优先级**：P1
- **描述**：管理后台创建/编辑模考试卷，配置题目组合、建议时长、考察范围。

### 3.3 会员计划管理
- **优先级**：P2
- **描述**：管理后台编辑会员套餐（价格、周期、权益列表），无需改代码上线。

### 3.4 数据看板
- **优先级**：P1
- **描述**：管理后台 Dashboard 展示核心运营指标：
  - 注册用户数、日活
  - 付费转化率、收入趋势
  - 题目完成率、模考参与率
  - 热门题库/专题排行

---

## 四、学习数据与激励

### 4.1 错题本
- **优先级**：P1
- **描述**：自动收集用户在练习和模考中答错的题目，支持按题库/专题筛选、重新练习。
- **要点**：
  - 错题自动入库（基于练习记录 + 模考成绩）
  - 错题列表与重新练习入口
  - 错题移除（用户确认已掌握）

### 4.2 学习计划 / 每日目标
- **优先级**：P2
- **描述**：用户设定每日学习目标（如每天练习 X 题），系统跟踪完成度并提醒。
- **要点**：
  - 目标设定入口（个人中心/首页）
  - 今日进度可视化（环形图/进度条）
  - 连续达标天数统计

### 4.3 成就系统
- **优先级**：P3
- **描述**：徽章、里程碑、连续打卡等游戏化激励。
- **示例成就**：
  - 首次练习、累计 100 题、连续 7 天打卡
  - 模考满分、全题库通关
  - 收藏达人、词汇大师

### 4.4 排行榜
- **优先级**：P3
- **描述**：按题库维度的学习排行榜（练习量、模考分数、连续天数），激发竞争。

## 五、通知与触达

### 5.1 移动端推送通知
- **优先级**：P2
- **描述**：iOS/Android 推送通知（每日学习提醒、新功能通知、会员到期提醒）。
- **技术栈**：Firebase Cloud Messaging (FCM) 或 APNs。

### 5.2 邮件通知
- **优先级**：P2
- **描述**：事务性邮件（注册欢迎、支付成功、会员到期提醒等）。

### 5.3 站内通知完善
- **优先级**：P2
- **描述**：关键事件自动生成站内通知（模考出分、会员开通、系统公告），无需管理员手动创建。
- **现状**：站内通知框架已就绪，但事件触发链路未建立。

### 6.1 学习成果分享
- **优先级**：P3

## 七、技术与基础设施

### 7.1 自动化测试
- **优先级**：P1
- **描述**：建立测试体系，覆盖关键路径。
- **范围**：
  - 后端：关键 Service 单元测试 + API E2E 测试
  - 前端：关键页面组件测试 + 用户流程 E2E（Playwright/Cypress）

### 7.2 API 文档（Swagger）
- **优先级**：P1
- **描述**：使用 `@nestjs/swagger` 自动生成 OpenAPI 文档，方便前后端联调和第三方对接。

### 7.3 接口限流与防护
- **优先级**：P1
- **描述**：
  - API 频率限制（防止刷接口）
  - 登录失败次数限制（防暴力破解）
  - 敏感操作二次验证

### 7.4 离线模式 / PWA
- **优先级**：P2
- **描述**：核心题库内容支持离线缓存，弱网/无网环境下仍可练习。

### 7.5 性能优化
- **优先级**：P2
- **描述**：
  - 前端代码分割（route-based lazy loading）
  - 图片/音频懒加载
  - 首屏加载优化
  - Bundle 体积分析

### 7.6 监控与日志
- **优先级**：P2
- **描述**：
  - 服务端错误监控（Sentry 或自建）
  - API 响应时间监控
  - 数据库慢查询日志
  - 前端异常捕获与上报

### 7.7 数据导出
- **优先级**：P2
- **描述**：用户可导出个人学习数据（练习记录、收藏、生词本等），符合 GDPR/个人信息保护法要求。

### 7.8 无障碍访问 (a11y)
- **优先级**：P3
- **描述**：满足 WCAG 2.1 AA 标准：
  - 键盘导航完整支持
  - 屏幕阅读器兼容
  - 色彩对比度合规
  - Focus 指示器清晰

## 八、产品体验优化

### 8.1 多语言内容支持
- **优先级**：P2
- **描述**：题库内容支持多语种（英语、日语、韩语等导游考试语种），不仅限于界面 i18n。
- **现状**：前端 i18n 框架已就绪，但题库内容仅有中文。

### 8.2 暗色模式完善
- **优先级**：P2
- **描述**：部分组件（图表 Recharts、波形图 wavesurfer、通知详情等）在暗色模式下显示不佳，需要适配。

### 8.3 首次引导 / Onboarding
- **优先级**：P2
- **描述**：新用户首次进入 App 时的引导流程（功能简介 + 题库绑定引导），降低上手门槛。

### 8.4 反馈/客服系统
- **优先级**：P3
- **描述**：应用内反馈入口（提交问题/建议）+ 客服对话（可先接入微信客服）。

### 8.5 学习小组 / 社区
- **优先级**：P3
- **描述**：按省份/语种创建学习小组，组内讨论、互相鼓励。

## 九、移动端专项

### 9.1 iOS 原生能力接入
- **优先级**：P1
- **描述**：通过 Capacitor 插件接入原生能力：
  - 音频录制权限
  - 推送通知
  - 应用内购买 (IAP)
  - 文件存储

### 9.2 Android 适配
- **优先级**：P2
- **描述**：当前仅有 iOS 原生项目，需补充 Android 平台支持。
