# 导游说系统后端与数据库实现文档（MVP）

本文用于配套 `导游说-需求文档.md`，给出可落地的后端 API、数据库模型、状态流与实现边界。  
原则：**功能严格对齐需求文档**，先满足核心闭环，再做增强。

---

## 1. 实现目标与边界

### 1.1 目标

- 支撑前端 5 个核心页面的真实数据与行为持久化：
  - 题库首页
  - 练习页
  - 模考页
  - 个人中心
  - 会员权益页
- 提供题库绑定配置、题目练习、收藏、生词、练习记录、模考记录等服务能力
- 题库绑定与用户学习数据以登录用户 `userId` 为 owner 持久化；前端可在设置中重新绑定题库

### 1.2 边界

MVP 不强制实现：

- 自动评分、AI 点评
- 录音上传与语音打分
- 复杂推荐系统

---

## 2. 后端技术架构建议

建议采用与现有 `echoon` 一致的 NestJS 模块化结构：

- `config-module`
- `question-bank-module`（题库、题型、题目）
- `practice-module`（练习会话、进度、快捷行为）
- `learning-asset-module`（收藏、生词）
- `mock-exam-module`（试卷、成绩）
- `profile-module`（概览、活跃度）
- `membership-module`（套餐、权益）
- `dictionary-module`（单词释义，可先 mock）

当前实现：**已登录用户 + `userId`** 作为所有用户域表（收藏、生词、练习进度、模考记录、绑定配置等）的外键 owner；接口通过 Better Auth session 解析 `userId`，不再使用匿名 `deviceId`。

## 3. 数据库设计（PostgreSQL + Prisma）

## 3.1 核心实体关系

### 题库域

- `ExamConfigOption`：省份、语种、考试类型、面试形式基础选项
- `QuestionBank`：题库主表（绑定配置维度）
- `QuestionTopic`：题型/专题（景点介绍、欢迎词等）
- `QuestionItem`：题目主数据（标题、难度、建议时长、关键词等）
- `QuestionContent`：题目正文（英文/中文、答案英文/中文、摘要）

### 学习行为域

- `UserBindingConfig`：用户当前绑定配置与题库
- `FavoriteQuestion`：收藏题
- `VocabularyWord`：生词本
- `PracticeRecord`：练习记录明细
- `PracticeProgress`：题目级学习进度（掌握度、已看/未看）

### 模考域

- `MockPaper`：模考试卷元信息（标准卷/强化卷）
- `MockPaperQuestion`：试卷题目关联
- `MockExamRecord`：模考成绩记录（分数、薄弱点）

### 用户看板域

- `DailyActivity`：活跃度热力图基础数据（日维度）
- `UserPreference`：偏好配置（自动播放等）
- `MembershipPlan` / `UserMembership`：会员套餐与用户订阅状态

## 3.2 建表建议（字段级）

以下为 MVP 必要字段建议（非完整 SQL）：

### `question_bank`

- `id`（pk）
- `name`
- `province`
- `language`
- `exam_type`
- `interview_form`
- `status`（active/inactive）
- `created_at` / `updated_at`

### `question_topic`

- `id`（pk）
- `bank_id`（fk）
- `code`（如 `scenic-intro`）
- `name`
- `sort_order`

### `question_item`

- `id`（pk）
- `topic_id`（fk）
- `title`
- `difficulty`（1~5）
- `suggested_duration_sec`
- `mastery_score`（0~100）
- `keywords`（jsonb/string[]）
- `focus_words`（jsonb/string[]）
- `is_favorite_count`（可选聚合）

### `question_content`

- `question_id`（pk, fk）
- `prompt_en`
- `prompt_zh`
- `answer_en`
- `answer_zh`
- `summary`

### `user_binding_config`

- `id`（pk）
- `user_id`（fk → `user`，唯一）
- `province`
- `language`
- `exam_type`
- `interview_form`
- `bank_id`（fk）
- `updated_at`

### `favorite_question`

- `id`（pk）
- `user_id`（fk → `user`）
- `question_id`（fk）
- `created_at`
- unique(`user_id`,`question_id`)

### `vocabulary_word`

- `id`（pk）
- `user_id`（fk → `user`）
- `term`
- `definition`（可空）
- `source_question_id`（可空 fk）
- `created_at`
- unique(`user_id`,`term`)

### `practice_record`

- `id`（pk）
- `user_id`（fk → `user`）
- `question_id`（fk）
- `action_type`（view/play/toggle_answer/toggle_translation/favorite/word_add）
- `payload`（jsonb）
- `created_at`

### `mock_exam_record`

- `id`（pk）
- `user_id`（fk → `user`）
- `paper_id`（fk）
- `score`
- `weakness`（jsonb/string[]）
- `taken_at`

### `user_preference`

- `id`（pk）
- `user_id`（fk → `user`，唯一）
- `auto_play`（bool）
- `language`（zh-CN/en）
- `theme`（light/dark/system）
- `updated_at`

## 3.3 与前端状态映射

- 题库绑定与 bootstrap：`GET /bootstrap` / `POST /config/bind` ↔ `user_binding_config`；前端用内存 store 反映 `configured`，不再使用 `guide-exam-config` localStorage
- `guide-exam-favorites` / `guide-exam-words`（前端 persist）↔ `favorite_question` / `vocabulary_word`（可按产品节奏做全量服务端同步）
- `guide-exam-preferences` ↔ `user_preference`（服务端按 `userId` 唯一）

## 4. API 设计（MVP）

建议统一前缀：`/api/v1/guide-exam`

## 4.1 路由与配置

- `GET /bootstrap`（需登录）
  - 返回当前绑定配置、题库摘要等；未绑定时 `configured=false`
- `GET /config/options`
  - 返回省份/语种/考试类型/面试形式选项
- `POST /config/bind`
  - 入参：4 个配置项
  - 出参：匹配题库与绑定结果
- `GET /config/current`
  - 返回当前绑定信息

## 4.2 题库首页

- `GET /question-bank/home`
  - 入参：`mode`、`keyword`（可选）
  - 出参：
    - Hero 数据
    - 景点介绍卡片列表
    - 其他题型统计与进度

## 4.3 练习页

- `GET /practice/topic/:topicId/questions`
  - 返回该 topic 的题目序列与基础信息
- `GET /practice/question/:questionId`
  - 返回题目内容（中英、答案、关键词、词汇）
- `POST /practice/action`
  - 记录用户行为（播放、答案显隐、翻译显隐、切题等）
- `GET /dictionary/lookup?term=xxx`
  - 返回词义（MVP 可先本地词典或 mock）

## 4.4 收藏与生词

- `GET /assets/favorites`
- `POST /assets/favorites/:questionId`
- `DELETE /assets/favorites/:questionId`
- `GET /assets/words`
- `POST /assets/words`
- `DELETE /assets/words/:term`

## 4.5 模考

- `GET /mock/papers`
- `GET /mock/recent-scores`
- `POST /mock/start`
- `POST /mock/submit`（MVP 可先只记录分数与薄弱点）

## 4.6 个人中心与会员

- `GET /profile/overview`
- `GET /profile/activity-heatmap`
- `GET /profile/practice-records`
- `GET /member/plans`
- `GET /member/current`

## 5. 业务规则落地（必须与需求一致）

### 5.1 首次绑定

- 首次访问若无 `user_binding_config`，`/bootstrap` 返回 `configured=false`
- 前端据此强制弹出绑定 Dialog

### 5.2 练习模式 vs 学习模式

- 练习模式：答案默认隐藏，按题独立控制显示状态
- 学习模式：答案默认显示，允许收起
- 建议将模式状态放在前端视图态，同时关键行为写 `practice_record`

### 5.3 自动播放与并发播放控制

- `user_preference.auto_play=true` 时，切题触发播放事件
- 后端仅记录行为；“同一时刻仅一个音频播放”主要由前端播放器控制

### 5.4 快捷键行为

快捷键属于前端交互层，但建议关键动作上报：

- `Space` -> `play_question`
- `A` -> `toggle_answer`
- `T` -> `toggle_translation`
- `F` -> `toggle_favorite`
- `←/→/Enter` -> `prev_or_next_question`

## 6. Prisma 模型组织建议

建议在 Prisma 中按领域分段并增加索引：

- 高频查询索引：
  - `question_item(topic_id, difficulty)`
  - `favorite_question(user_id, created_at desc)`
  - `vocabulary_word(user_id, created_at desc)`
  - `practice_record(user_id, created_at desc)`
  - `mock_exam_record(user_id, taken_at desc)`

并保证以下唯一约束：

- `favorite_question(user_id, question_id)`
- `vocabulary_word(user_id, term)`
- `user_binding_config(user_id)`（一人一份当前配置）
- `user_preference(user_id)`（一人一份偏好）

## 7. 数据初始化（Seed）建议

MVP 需要一批稳定样例数据：

- 至少 2 个省份 x 2 语种 x 2 面试形式组合
- 每个题型至少 10~20 题
- 模考试卷至少 2 套（标准/强化）
- 最近成绩至少 5~10 条样例（便于页面展示）

初始化可分 3 个 seed 文件：

1. `seed-config.ts`（配置与题库）
2. `seed-questions.ts`（题目与内容）
3. `seed-user-sample.ts`（收藏/生词/记录/成绩）

---

## 8. 安全与非功能建议（MVP）

- 输入校验：DTO + `zod` 双层校验（前后端）
- 限流：`/dictionary/lookup` 与行为上报接口做基础限流
- 审计：关键写操作记录 `requestId/userId`
- 性能：
  - 首页搜索接口做 debounce（前端）+ 索引（后端）
  - 列表接口支持分页与轻量字段

---

## 9. 交付清单

后端与数据库 MVP 交付建议按以下顺序：

1. Prisma schema + migration
2. seed 数据
3. 配置/题库/练习只读接口
4. 收藏/生词/偏好写接口
5. 模考与个人中心聚合接口
6. 前后端联调与需求逐条验收

---

## 10. 验收标准（与需求文档对齐）

- 需求文档 4.x 所有功能点均有对应 API 或前端可依赖数据源
- 需求文档 5.x 业务规则可被验证（配置、偏好、行为记录）
- 前端 5 个主路由可在真实接口数据下跑通
- 收藏、生词、偏好、练习记录至少具备“增删查”能力

达到以上标准，即可认定“后端与数据库 MVP 可支撑当前导游说系统”。
